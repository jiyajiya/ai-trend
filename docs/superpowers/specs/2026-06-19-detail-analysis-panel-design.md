# 상세 분석 패널 (Detail Analysis Panel) — 설계

작성일: 2026-06-19
대상 프로젝트: AI 트렌드 대시보드 (`ai-trend`)

## 배경 / 문제

현재 대시보드는 **사이드바 + 리더(단일 컬럼)** 구조로, 각 항목을 카드로 보여주며 `summaryKo`(2~3문장, ~200자)만 표시한다.

영상 요약 단계에서 `watch` 스킬이 자막 트랜스크립트를 받아 분석하지만, **최종적으로 저장되는 건 짧은 `summaryKo`뿐**이고 트랜스크립트·심화 분석은 버려진다. (영상 `rawText`는 제목만 들어있음.) 따라서 "영상을 분석했으니 전체 내용이 있을 것"이라는 기대가 데이터상으로는 성립하지 않는다.

사용자는 카드를 클릭하면 **우측 패널에 영상/글의 전체 내용을 분석한 결과**를 보고 싶어 한다. 이를 위해서는 요약 단계에서 **더 긴 구조화 분석을 새 필드로 저장**하고, 이를 보여줄 **3번째 우측 컬럼 패널**을 추가해야 한다.

## 결정 사항 (확정)

| 항목 | 결정 |
|---|---|
| 분석 내용 | 구조화된 상세 분석 (핵심 포인트 · 챕터별 정리 · 인용) |
| 저장 형태 | **구조화 객체** (마크다운 문자열 아님 — 파서 불필요·테스트 용이) |
| 화면 배치 | **3번째 우측 컬럼 패널** (모바일은 전체 오버레이) |
| 적용 범위 | `viewType` 기준 **video · news · blog** (SNS 제외: Hacker News·GeekNews·Reddit·Threads) |
| 기존 데이터 | **백필** — 기존 영상 24건 + 뉴스/블로그 항목에 `analysis` 채움 |

## 데이터 모델 — 새 필드 `analysis`

각 아이템(대상 타입)에 선택적 필드 `analysis`를 추가한다.

```jsonc
"analysis": {
  "points":   ["핵심 포인트 불릿 (3~6개)"],
  "sections": [
    { "heading": "챕터/주제명", "body": "2~4문장 정리" }
  ],
  "quotes":   ["주목할 인용·수치 (0~3개)"]   // 선택, 없으면 생략 또는 빈 배열
}
```

규칙:
- `analysis`가 없으면(생성 실패/비대상) 필드를 **생략**한다. 뷰는 "상세 분석 없음" + 기존 `summaryKo`로 폴백.
- `points`는 최소 1개 이상일 때만 의미가 있다. 모두 비면 `analysis` 생략.
- 트랜스크립트 원문은 저장하지 않는다(용량). `analysis`만 저장.
- 기존 필드(`summaryKo`, `tags`, `entities`, `cats`, `summaryStatus`)는 그대로 유지. `analysis`는 **추가**일 뿐 기존 카드 요약을 대체하지 않는다.

## 파이프라인 변경 (`pipeline/RUNBOOK.md`)

요약(summarize) 단계 규칙에 `analysis` 생성을 추가한다.

- **영상(youtube)**: `watch` 트랜스크립트로 `summaryKo`를 만들 때, **동일 입력**으로 `analysis`(points/sections/quotes)까지 생성한다. 자막이 없어 fallback인 경우 `analysis` 생략.
- **뉴스/블로그(news·blog)**: `rawText`(필요 시 `url` WebFetch 보강)로 `analysis` 생성. 본문이 빈약하면 `analysis` 생략 허용.
- **SNS·repo·model**: `analysis` 생성하지 않음.

생성 분량 가이드: `points` 3~6개, `sections` 2~5개(각 2~4문장), `quotes` 0~3개. 한국어.

`pipeline/merge.mjs`는 `analysis`를 **그대로 통과·보존**한다. (기존 항목 병합 시 새로 들어온 `analysis`가 있으면 갱신, 없으면 기존 값 유지.)

## UI 변경 (`web/`)

### 레이아웃
```
+--------+------------------+------------------+
| 사이드 | 피드(카드 목록)   | 상세 분석 패널    |
| 바     | [● 선택된 카드]  | 핵심 포인트       |
|        | [ 카드 ]         | • …              |
|        | [ 카드 ]         | 챕터별 정리       |
|        |                  | 인용·수치         |
+--------+------------------+------------------+
```
- 데스크톱(넓은 폭): `사이드바 | 피드 | 분석패널` 3컬럼. 패널 폭 고정(예: 360~420px).
- 모바일/좁은 폭: 패널이 피드 위 **전체 오버레이**로 전환, 우상단 닫기(×) 버튼.
- 아무 카드도 선택되지 않은 초기 상태: 패널은 안내 문구(예: "카드를 선택하면 상세 분석이 여기에 표시됩니다") 또는 데스크톱에서 숨김.

### 상태 / 상호작용
- `state.selectedId` 추가. 카드 클릭 → `selectedId` 설정 → 패널 렌더.
- 같은 카드 재클릭 또는 닫기 버튼 → `selectedId = null` → 패널 닫힘.
- 피드 전환·필터 변경으로 선택 카드가 목록에서 사라지면 `selectedId` 해제.
- 북마크(★) 버튼 클릭은 카드 선택과 **분리**(이벤트 전파 차단)되어 패널을 열지 않는다.

### 패널 내용 (순서)
1. 헤더: 제목 · 출처 · 시간 · 원문 링크(새 탭)
2. 핵심 포인트 (`points`) — 불릿 리스트
3. 챕터별 정리 (`sections`) — 소제목 + 본문
4. 인용·수치 (`quotes`) — 있으면 인용 블록
5. `analysis` 없을 때: "상세 분석 없음" 안내 + `summaryKo` 표시

### 어댑터 (`web/adapt.mjs`)
- `toViewItem`에 `analysis` 전달 추가. 정규화 함수로 잘못된 형태(누락 필드, 비배열)를 안전하게 보정:
  - `points`/`sections`/`quotes`가 배열이 아니면 빈 배열.
  - `sections` 항목에 `heading`/`body` 누락 시 빈 문자열로 보정.
  - 유효 내용이 전혀 없으면 `analysis: null`로 정규화(뷰가 폴백 처리).
- 렌더(HTML 생성)는 `app.js`에서 수행하며, 모든 텍스트는 기존 `esc()`로 이스케이프한다.

## 백필 (1회성)

기존 `data/feed.json`의 대상 항목(영상 24 + 뉴스/블로그)에 `analysis`를 채운다.

- 영상: `url`/`videoId`로 `watch` 재실행 → `analysis` 생성.
- 뉴스/블로그: 저장된 `rawText` 사용(필요 시 `url` 보강).
- SNS 항목은 제외.
- 결과를 정상 파이프라인(summarize → merge)으로 반영하고 push하여 GitHub Pages 갱신.
- 백필은 사용량·시간을 소모하므로 **한 번만** 실행. 이후에는 신규 항목이 정상 파이프라인에서 `analysis`를 얻는다.

## 테스트 (`test/`)

- `adapt.mjs` 분석 정규화: `analysis` 정상 / 없음 / 일부 필드 누락 / 비배열 입력 케이스.
- `merge.mjs`: 병합 시 `analysis` 보존(신규 있으면 갱신, 없으면 기존 유지) 확인.
- 뷰어 어댑터: `toViewItem`이 `analysis`를 정규화해 전달하는지.

## 영향 범위 / 리스크

- 수정 파일: `pipeline/RUNBOOK.md`, `pipeline/merge.mjs`, `web/adapt.mjs`, `web/app.js`, `web/style.css`, `test/*`.
- 데이터: `data/feed.json`(백필로 `analysis` 추가) — 게시 데이터 용량 증가(항목당 수백 byte~1KB).
- 리스크:
  - 백필이 사용량/시간을 소모(영상 watch 재실행 24건).
  - 분석 품질은 LLM 요약에 의존 — RUNBOOK 가이드로 분량·구조 일관성 확보.
  - 3컬럼 추가로 좁은 화면 레이아웃 검증 필요(반응형 오버레이).
